## 一、你的真实需求

**不是数据分析，是视频内容情报提取：**

每2小时：获取B站各分区热门/增长最快视频 → 把UP主说的话转成文字 → AI提炼最新动态情报。

## 二、B站API实测结论（2026.2.11真实调用）

### 能用的

| 接口 | 状态 | 返回内容 |
|------|------|---------|
| `ranking/v2?rid=0` | ✅ | 全站Top100视频（bvid/标题/播放量/UP主/分区/时长） |
| `ranking/region?rid=188&day=3` | ✅ | 旧版分区排行（数码区等） |
| `popular?ps=20&pn=1` | ✅ | 热门推荐 |

### 不能用的

| 接口 | 问题 |
|------|------|
| `ranking/v2?rid=188` | ❌ -352风控拦截 |
| `player/v2` 字幕 | ❌ `need_login_subtitle:true`，未登录`subtitles`永远为空 |
| `popular/series` | ❌ -352风控 |

### 重大事件

**2026年1月28日**，B站向 `bilibili-API-collect` 发律师函，仓库已永久关停。B站明确认为这些API是"非公开接口"，有主动执法意愿。

### 结论

**官方API只能满足约20%的需求**（发现视频）。核心80%（获取视频内容文字）完全无法满足。你的直觉是对的。

## 三、技术方案：视频→文字 的完整链路

```
发现视频（API/Playwright）
    ↓
获取内容（CC字幕优先 → Whisper兜底）
    ↓
AI分析（DeepSeek提炼情报）
    ↓
推送（飞书/Email/Telegram）
```

### 第1层：发现视频
- B站公开API（ranking/popular）
- Playwright渲染热门页面（备用）
- newsnow已有B站数据源（MIT协议）

### 第2层：视频→文字（核心）

**路径A - CC字幕提取（优先，覆盖约60%视频）：**
- 用自己B站账号的Cookie调`player/v2` → `subtitles` → 下载字幕JSON
- 成本：¥0

**路径B - 音频+Whisper转写（兜底，覆盖剩余40%）：**
- yt-dlp下载音频 → faster-whisper本地转写
- CPU跑：10分钟音频≈4分钟转写，每周期20个视频≈80分钟，2小时内完成
- 成本：¥0（电费忽略）

**两者结合 = 100%覆盖。**

### 第3层：AI分析
- DeepSeek V3：¥9/月（1000条×200字输入 + 50条分析输出）

### 第4层：推送
- 飞书Webhook / Email / Telegram Bot

## 四、关于MediaCrawler

你说"参照原理用AI重写一个"——**完全可行且合法**。

版权保护的是代码表达，不是设计思想。你用到的都是通用技术：
- Playwright驱动浏览器（开源工具）
- Cookie保存/携带（标准Web技术）
- yt-dlp下载音频（开源工具）

**但你其实不需要整个MediaCrawler。** 它支持7个平台N种功能，你只需要"B站热门视频→文字"这一条线。全部代码约750行Python，5天能写完：

| 模块 | 代码量 | 耗时 |
|------|-------|------|
| Cookie管理（Playwright扫码登录） | ~100行 | 1天 |
| 热门视频发现（API+分区筛选） | ~150行 | 半天 |
| CC字幕提取 | ~80行 | 半天 |
| 音频下载（yt-dlp） | ~50行 | 2h |
| Whisper转写（faster-whisper） | ~60行 | 2h |
| DeepSeek分析 | ~100行 | 1天 |
| 推送+定时调度+增量检测 | ~210行 | 1.5天 |
